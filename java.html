What is Java?

Java is a powerful, object-oriented programming language that runs on billions of devices worldwide. It's known for its "Write Once, Run Anywhere" capability.

🚀 Key Features:

  • Platform Independent: Java code compiles to bytecode that runs on any Java Virtual Machine (JVM)
  • Object-Oriented: Everything in Java is an object
  • Secure: Built-in security features and no pointers
  • Multithreaded: Can perform multiple tasks simultaneously

Your First Java Program

// Every Java program must have a class
public class HelloWorld {
    // main() is the entry point of every Java application
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

Object-Oriented Programming (OOP)

OOP is a programming paradigm that organizes software design around objects rather than functions and logic.

🎯 Four Pillars of OOP:

  1. Encapsulation: Bundling data and methods that work on that data within one unit
  2. Inheritance: Creating new classes from existing ones
  3. Polymorphism: One name, many forms
  4. Abstraction: Hiding complex reality while exposing only essentials

Real-World Analogy

Think of a Car as a class. Different car models (Toyota, BMW) are objects of that class. They inherit basic car properties but have their own unique features.

Classes and Objects

A class is a blueprint for creating objects. An object is an instance of a class.

// Class definition
public class Student {
    // Fields (attributes)
    String name;
    int age;
    double gpa;

    // Constructor
    public Student(String name, int age, double gpa) {
        this.name = name;
        this.age = age;
        this.gpa = gpa;
    }

    // Method
    public void displayInfo() {
        System.out.println("Name: " + name + ", Age: " + age + ", GPA: " + gpa);
    }
}

Inheritance

Inheritance allows a class to inherit properties and methods from another class. The class being inherited from is called the parent/superclass, and the class that inherits is called the child/subclass.

// Parent class
class Animal {
    String name;
    
    public void eat() {
        System.out.println("This animal eats food.");
    }
}

// Child class inheriting from Animal
class Dog extends Animal {
    public void bark() {
        System.out.println("Woof! Woof!");
    }
}

🔗 Types of Inheritance:

  • Single: One class extends another class
  • Multilevel: Class A → Class B → Class C
  • Hierarchical: One class extended by multiple classes
  • Multiple: Not supported in Java (use interfaces instead)

Polymorphism

Polymorphism means "many forms". It allows methods to do different things based on the object that is acting upon them.

Method Overloading (Compile-time Polymorphism)

class Calculator {
    // Same method name, different parameters
    int add(int a, int b) {
        return a + b;
    }
    
    double add(double a, double b) {
        return a + b;
    }
}

Method Overriding (Runtime Polymorphism)

class Animal {
    public void makeSound() {
        System.out.println("Animal makes a sound");
    }
}

class Cat extends Animal {
    @Override
    public void makeSound() {
        System.out.println("Meow!");
    }
}

Encapsulation

Encapsulation is about bundling data and methods together and restricting direct access to some of the object's components.

public class BankAccount {
    // Private fields - cannot be accessed directly
    private String accountNumber;
    private double balance;

    // Public methods to access private fields
    public String getAccountNumber() {
        return accountNumber;
    }

    public double getBalance() {
        return balance;
    }

    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
        }
    }
}

🎯 Benefits of Encapsulation:

  • Data Hiding: Internal implementation is hidden
  • Flexibility: Can change internal implementation without affecting users
  • Reusability: Easy to reuse encapsulated code
  • Testing: Easier to unit test

Methods in Java

Methods are blocks of code that perform specific tasks. They make code reusable and organized.

Method Type Description Example
Instance Method Belongs to object, can access instance variables public void display() { }
Static Method Belongs to class, called using class name public static void main() { }
Abstract Method No implementation, must be overridden abstract void calculate();
Final Method Cannot be overridden by subclasses public final void process() { }

Collections Framework

Collections are used to store and manipulate groups of objects. Java provides a rich collections framework.

import java.util.ArrayList;

ArrayList<String> fruits = new ArrayList<>();
fruits.add("Apple");
fruits.add("Banana");
fruits.add("Orange");

for (String fruit : fruits) {
    System.out.println(fruit);
}

Exception Handling

Exceptions are unexpected events that occur during program execution. Java provides robust exception handling.

try {
    int result = 10 / 0; // This will cause ArithmeticException
} catch (ArithmeticException e) {
    System.out.println("Cannot divide by zero!");
} finally {
    System.out.println("This always executes");
}

  • NumberFormatException: Invalid number conversion
  • IOException: Input/output operations failure
