What is a Database?

A database is an organized collection of data stored and accessed electronically. Think of it as a digital filing cabinet where you can store, organize, and retrieve information efficiently.

📁 Simple Definition:

A database is like a super-organized digital library where instead of books, you store data that computers can quickly find and use.

Types of Databases

Different databases are designed for different purposes. The main categories are relational (SQL) and non-relational (NoSQL).

Relational Databases (SQL)

🏗️ Structured like Excel Tables

Data is organized in tables with rows and columns, and tables can be linked together using relationships.

Examples: MySQL, PostgreSQL, Oracle, SQL Server

NoSQL Databases

🎯 Flexible Data Structures

Data can be stored in various formats - documents, key-value pairs, graphs, or wide columns.

Examples: MongoDB, Redis, Cassandra, Neo4j

SQL Basics

SQL (Structured Query Language) is the standard language for communicating with relational databases.

🗣️ Database Language

SQL is how we talk to databases. It's like giving instructions to a very organized librarian who can instantly find exactly what you need.

Basic SQL Commands

SELECT name, email FROM users WHERE age > 18;

This means: "Show me the names and emails of all users who are older than 18"

Database Design

Good database design is crucial for building efficient and maintainable applications.

🏗️ Blueprint for Data

Database design is like architectural planning for your data. You decide what information to store and how it connects before building anything.

Design Principles

  • Keep it simple: Don't overcomplicate the design
  • Plan for growth: Design should handle future data
  • Use meaningful names: Table and column names should be clear
  • Avoid data duplication: Store each piece of information once
  • Consider performance: Design for efficient queries

Foreign Keys

Foreign keys are how we create relationships between tables:

-- Customers table
CREATE TABLE customers (
  customer_id INT PRIMARY KEY,
  name VARCHAR(100)
);

-- Orders table with foreign key
CREATE TABLE orders (
  order_id INT PRIMARY KEY,
  customer_id INT,
  order_date DATE,
  FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
);

This ensures that every order must belong to a valid customer!

Database Transactions

Transactions ensure that database operations are completed reliably, following the ACID principles.

ACID Properties

Click to learn about ACID properties

Transaction Example: Bank Transfer

Imagine transferring $100 from Alice to Bob:

BEGIN TRANSACTION;

-- 1. Deduct $100 from Alice's account
UPDATE accounts SET balance = balance - 100
WHERE account_id = 'Alice';

-- 2. Add $100 to Bob's account
UPDATE accounts SET balance = balance + 100
WHERE account_id = 'Bob';

COMMIT;

Atomicity ensures: Either both updates happen, or neither happens. No half-completed transfers!

Why Transactions Matter

  • Data Integrity: Prevents partial updates
  • Error Recovery: Can rollback on failure
  • Concurrency: Multiple users can work safely
  • Reliability: Data remains consistent
