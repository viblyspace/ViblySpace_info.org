Chapter 1: Introduction to CSS

What is CSS?

CSS (Cascading Style Sheets) is a stylesheet language used to describe the presentation of documents written in HTML or XML. CSS describes how elements should be rendered on screen, on paper, in speech, or on other media.

CSS is one of the cornerstone technologies of the World Wide Web, alongside HTML and JavaScript. While HTML provides the structure of a webpage, CSS controls the visual presentation.

Teaching Tip: Use the analogy of a house - HTML is the structure (walls, floors, rooms), while CSS is the decoration (paint, furniture, lighting).

CSS allows you to control:

  • Colors and backgrounds
  • Fonts and text formatting
  • Layout and positioning
  • Spacing and dimensions
  • Animations and transitions
  • Responsive design for different devices

Brief History of CSS

CSS was first proposed by Håkon Wium Lie in 1994 while working at CERN with Tim Berners-Lee. The evolution of CSS:

  • 1996: CSS Level 1 (CSS1) - First official specification
  • 1998: CSS Level 2 (CSS2) - Added positioning, media types
  • 2011: CSS 2.1 - Fixed errors and became a standard
  • 2011+: CSS3 - Modular approach with separate specifications

CSS3 introduced many new features that made web design more powerful:

  • Media queries for responsive design
  • Flexbox and Grid for advanced layouts
  • Transforms, transitions, and animations
  • Custom properties (CSS variables)
  • Advanced selectors
  • Multiple background images

How CSS Works

CSS works by associating rules with HTML elements. These rules govern how the content of specified elements should be displayed.

The browser follows these steps when rendering a webpage with CSS:

  1. Loads the HTML and creates the DOM (Document Object Model)
  2. Loads the CSS and parses it
  3. Creates the CSSOM (CSS Object Model)
  4. Combines DOM and CSSOM to create a render tree
  5. Lays out the elements on the page
  6. Paints the final visual representation

Key Concepts for Students

  • Separation of concerns - HTML for structure, CSS for presentation
  • Cascade - How CSS resolves conflicts when multiple rules apply
  • Specificity - How browsers decide which CSS rule takes precedence
  • Inheritance - How some properties pass from parent to child elements
  • The Box Model - Fundamental concept of how elements are sized and spaced

Common misconceptions to address:

  • CSS is not a programming language but a declarative language
  • CSS can't add content to a page, only style existing content
  • CSS is not just about making things "look pretty" - it's crucial for usability and accessibility

Chapter 3: CSS Box Model

Understanding the Box Model

Every element in CSS is represented as a rectangular box. The CSS box model describes how these boxes are sized and spaced.

The box model consists of:

  • Content - The actual content of the box (text, images, etc.)
  • Padding - Clears an area around the content (inside the box)
  • Border - A border that goes around the padding and content
  • Margin - Clears an area outside the border (between boxes)

Teaching Tip: Use the analogy of a picture frame: the content is the picture, padding is the matting, border is the frame, and margin is the space between frames on a wall.

Margin Collapsing

Margin collapsing is a behavior where the vertical margins of adjacent elements collapse into a single margin:

Margin Collapsing Example
.box1
{
  margin-bottom
:
20px
;
}

.box2
{
  margin-top
:
30px
;
}
/* The space between box1 and box2 will be 30px (not 50px) */

When margin collapsing occurs:

  • Adjacent sibling elements
  • Parent and first/last child elements (when there's no border, padding, or content separating them)
  • Empty blocks (with no content, padding, or border)

How to prevent margin collapsing:

  • Add padding or border to the parent element
  • Use flexbox or grid layout
  • Add overflow: auto or overflow: hidden to the parent

Chapter 10: Responsive Design

Introduction to Responsive Design

Responsive web design (RWD) is an approach to web design that makes web pages render well on a variety of devices and window or screen sizes.

Key principles of responsive design:

  • Fluid Grids - Layouts that use relative units like percentages
  • Flexible Images - Images that scale within their containing elements
  • Media Queries - CSS technique to apply different styles based on device characteristics
Viewport Meta Tag
<!-- Essential for responsive design -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">

Important: The viewport meta tag tells the browser how to control the page's dimensions and scaling. Without it, mobile devices will render the page at desktop width and scale it down.

Media Queries

Media queries allow you to apply CSS styles depending on device characteristics:

;

/* Orientation-based */

{
  .sidebar

:

;
  }

Chapter 13: CSS Variables (Custom Properties)

Introduction to CSS Variables

CSS variables (custom properties) allow you to store specific values for reuse throughout a document. They follow the same inheritance and cascade rules as other CSS properties.

CSS Variable Basics
/* Declaring CSS variables */
:root
{
  --primary-color
:
#3498db
;
  --secondary-color
:
#2ecc71
;
  --font-size
:
16px
;
  --spacing
:
1rem
;
}

/* Using CSS variables */
.button
{
  background-color
:
var(--primary-color)
;
  font-size
:
var(--font-size)
;
  padding
:
var(--spacing)
;
}

/* With fallback values */
.heading
{
  color
:
var(--heading-color, #333)
;
}

Teaching Tip: Compare CSS variables to variables in programming languages - they store values that can be reused and easily changed throughout your stylesheet.
