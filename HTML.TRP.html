Chapter 1: Introduction to HTML

What is HTML?

HTML (HyperText Markup Language) is the standard markup language for creating web pages and web applications. It describes the structure of a web page using a system of tags and attributes.

HTML is not a programming language; it doesn't have logic or computational capabilities. Instead, it's a markup language that tells web browsers how to display content.

Teaching Tip: Start by comparing HTML to the skeleton of a human body - it provides the basic structure that everything else builds upon. CSS would be the skin and clothing, while JavaScript would be the muscles and nervous system.

HTML documents are plain text files that contain:

  • Text content
  • Markup tags that define structure and presentation
  • References to other resources like images, CSS, and JavaScript

Brief History of HTML

HTML was created by Tim Berners-Lee in 1991 while working at CERN. The evolution of HTML:

  • 1991: HTML first proposed with 18 elements
  • 1995: HTML 2.0 - First standardized version
  • 1997: HTML 3.2 - Added tables, applets, text flow
  • 1999: HTML 4.01 - Major cleanup and standardization
  • 2000: XHTML 1.0 - XML-based version
  • 2014: HTML5 - Current standard with multimedia and semantic elements

HTML5 introduced many new features that made web development more powerful:

  • New semantic elements (header, footer, article, section, etc.)
  • Native multimedia support (audio, video)
  • Canvas for drawing graphics
  • Local storage capabilities
  • Improved form controls

How HTML Works

HTML documents are plain text files saved with .html extension. When a browser opens an HTML file, it reads the tags and renders the content accordingly.

The browser follows these steps:

  1. Parses the HTML to create a Document Object Model (DOM)
  2. Processes any CSS to create a CSS Object Model (CSSOM)
  3. Combines DOM and CSSOM to create a render tree
  4. Lays out the elements on the page
  5. Paints the final visual representation
<!-- Basic HTML Document Structure -->
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>
My First Web Page
</title>
</head>
<body>
  <h1>
Welcome to My Website
</h1>
  <p>
This is my first web page created with HTML.
</p>
</body>
</html>

Teaching Strategy: Encourage students to view the source code of their favorite websites (right-click > View Page Source) to see real-world HTML in action.

Key Concepts for Students

  • HTML is a markup language, not a programming language - It structures content but doesn't perform calculations or logic
  • Tags define elements - Most elements have opening and closing tags
  • Elements can be nested - Tags can contain other tags
  • Attributes provide additional information - Added to opening tags
  • Browsers are forgiving - They try to display content even with errors

Common misconceptions to address:

  • HTML is not "programming" in the traditional sense
  • HTML alone cannot create interactive websites
  • HTML structure is separate from presentation (CSS) and behavior (JavaScript)

Tools for HTML Development

Students need minimal tools to start learning HTML:

  • Text Editors: VS Code, Sublime Text, Atom, or even Notepad
  • Web Browsers: Chrome, Firefox, Safari, Edge for testing
  • Developer Tools: Built into modern browsers for debugging

Recommended setup for beginners:

  1. Install VS Code (free and feature-rich)
  2. Install the Live Server extension for real-time preview
  3. Use Chrome or Firefox with developer tools

Chapter 2: HTML Basics & Structure

Document Structure Essentials

Every valid HTML document must include these essential components:

<!-- Required for HTML5 -->
<!DOCTYPE html>
<!-- Root element with language attribute -->
<html lang="en">
<!-- Contains metadata -->
<head>
  <!-- Character encoding -->
  <meta charset="UTF-8">
  <!-- Responsive viewport -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <!-- Page title (appears in browser tab) -->
  <title>
Page Title
</title>
</head>
<!-- Visible content -->
<body>
  <!-- All visible elements go here -->
</body>
</html>

Explanation of each component:

  • <!DOCTYPE html> - Declares the document type and version of HTML
  • <html lang="en"> - Root element with language attribute for accessibility
  • <head> - Contains metadata, links to resources, and the page title
  • <meta charset="UTF-8"> - Ensures proper character encoding
  • <meta name="viewport"> - Makes page responsive on mobile devices
  • <title> - Appears in browser tab and search results
  • <body> - Contains all visible page content

Understanding HTML Elements

HTML elements are the building blocks of web pages. Each element consists of:

Component Description Example
Opening Tag Starts the element, may include attributes <p>
Content The actual content between tags This is text content
Closing Tag Ends the element (note the forward slash) </p>
Empty Element Elements with no content, self-closing <br> or <img src="image.jpg">

Important: HTML tags are case-insensitive, but lowercase is the standard convention. Always use lowercase for consistency and readability.

Chapter 3: Text Formatting in HTML

Text-Level Semantic Elements

HTML provides semantic elements to add meaning to text content:

<!-- Strong importance -->
<strong>
This text is important
</strong>

<!-- Emphasis -->
<em>
This text is emphasized
</em>

<!-- Small text -->
<small>
This is fine print
</small>

<!-- Marked/highlighted text -->
<mark>
This text is highlighted
</mark>

<!-- Deleted text -->
<del>
This text has been deleted
</del>

<!-- Inserted text -->
<ins>
This text has been inserted
</ins>

Why use semantic elements:

  • Better accessibility for screen readers
  • Improved SEO (Search Engine Optimization)
  • Clearer code meaning for developers
  • Future-proof styling (presentation can be changed with CSS)

Presentational Elements (Use Sparingly)

These elements affect appearance but don't add semantic meaning:

<!-- Bold (use strong instead for importance) -->
<b>
This text is bold
</b>

<!-- Italic (use em instead for emphasis) -->
<i>
This text is italic
</i>

<!-- Underline (avoid for links) -->
<u>
This text is underlined
</u>

Best Practice: Avoid using presentational elements when semantic elements are available. Use CSS for presentation instead of HTML.

When to use presentational elements:

  • <b> - When you want to draw attention without implying importance
  • <i> - For technical terms, foreign phrases, or thoughts
  • <u> - For proper names in Chinese or to indicate spelling errors

Chapter 4: Lists in HTML

Unordered Lists

Unordered lists are used when the order of items doesn't matter:

<!-- Basic unordered list -->
<ul>
  <li>
First item
</li>
  <li>
Second item
</li>
  <li>
Third item
</li>
</ul>

Browser default styling for unordered lists:

  • Bullet points (discs) for top-level items
  • Circles for nested lists
  • Squares for deeply nested lists

Teaching Tip: Explain that the default bullet style can be changed with CSS using the list-style-type property (disc, circle, square, none).

Ordered Lists

Ordered lists are used when the sequence of items is important:

<!-- Basic ordered list -->
<ol>
  <li>
First step
</li>
  <li>
Second step
</li>
  <li>
Third step
</li>
</ol>

Ordered list attributes:

<!-- Start from specific number -->
<ol start="5">
  <li>
Item 5
</li>
  <li>
Item 6
</li>
</ol>

<!-- Reverse numbering -->
<ol reversed>
  <li>
Item 3
</li>
  <li>
Item 2
</li>
  <li>
Item 1
</li>
</ol>

Numbering styles can be controlled with CSS using the list-style-type property:

  • decimal (1, 2, 3)
  • lower-roman (i, ii, iii)
  • upper-roman (I, II, III)
  • lower-alpha (a, b, c)
  • upper-alpha (A, B, C)

Description Lists

Description lists are used for terms and their descriptions:

<!-- Description list -->
<dl>
  <dt>
HTML
</dt>
  <dd>
HyperText Markup Language - the standard markup language for web pages.
</dd>

  <dt>
CSS
</dt>
  <dd>
Cascading Style Sheets - used for styling web pages.
</dd>

  <dt>
JavaScript
</dt>
  <dd>
A programming language that enables interactive web pages.
</dd>
</dl>

Common uses for description lists:

  • Glossaries and dictionaries
  • Metadata (key-value pairs)
  • FAQ sections
  • Any list of terms with descriptions

Nested Lists

Lists can be nested inside other lists to create hierarchical structures:

<!-- Nested list example -->
<ul>
  <li>
Fruits
    <ul>
      <li>
Apples
</li>
      <li>
Oranges
</li>
    </ul>
  </li>
  <li>
Vegetables
    <ul>
      <li>
Carrots
</li>
      <li>
Broccoli
</li>
    </ul>
  </li>
</ul>

Important: Pay attention to proper indentation when creating nested lists. Each nested list should be inside an <li> element of its parent list.

You can mix different types of lists:

<!-- Mixed list types -->
<ol>
  <li>
Main step
    <ul>
      <li>
Substep 1
</li>
      <li>

</li>

Chapter 5: Links & Navigation

The Anchor Element

The <a> (anchor) element creates hyperlinks:

<!-- Basic link -->
<a href="https://www.example.com">
Visit Example.com
</a>

<!-- Link opening in new tab -->
<a href="https://external-site.com" target="_blank">
External Site
</a>

<!-- Email link -->
<a href="mailto:contact@example.com">
Email Us
</a>

<!-- Telephone link -->
<a href="tel:+1234567890">
Call Us
</a>

Common link attributes:

Attribute Purpose Example
href Specifies the URL href="page.html"
target Specifies where to open the link target="_blank"
title Additional information (tooltip) title="Learn more"
rel Relationship between documents rel="nofollow"
download Prompts to download the linked resource download="filename"

Security Consideration: Use rel="noopener noreferrer" with target="_blank" to prevent potential security vulnerabilities.

Navigation Menus

The <nav> element defines a section of navigation links:

<!-- Semantic navigation -->
<nav>
  <ul>
    <li>
<a href="index.html">
Home
</a>
</li>
    <li>
<a href="about.html">
About
</a>
</li>
    <li>
<a href="contact.html">
Contact
</a>
</li>
  </ul>
</nav>

Best practices for navigation:

  • Use semantic <nav> element for major navigation blocks
  • Structure navigation with lists for better accessibility
  • Provide clear, descriptive link text
  • Include a "skip to main content" link for accessibility
  • Ensure navigation is usable without a mouse

Link Types and URL Schemes

HTML supports various URL schemes for different purposes:

<!-- Absolute URL -->
<a href="https://www.example.com/page.html">
Absolute Link
</a>

<!-- Relative URL -->
<a href="about.html">
Relative Link
</a>
<a href="../parent/page.html">
Parent Directory Link
</a>

<!-- Same page anchor -->
<a href="#section1">
Jump to Section 1
</a>

<!-- File download -->
<a href="document.pdf" download>
Download PDF
</a>

Special URL schemes:

  • mailto: - Opens default email client
  • tel: - Initiates phone call on mobile devices
  • sms: - Opens SMS messaging
  • ftp: - Links to FTP servers
  • javascript: - Executes JavaScript code (use sparingly)

Internal Page Navigation

You can create links to specific sections within the same page using ID attributes:

<!-- Navigation to sections -->
<nav>
  <ul>
    <li>
<a href="#intro">
Introduction
</a>
</li>
    <li>
<a href="#features">
Features
</a>
</li>
    <li>
<a href="#contact">
Contact
</a>
</li>
  </ul>
</nav>

<!-- Sections with IDs -->
<section id="intro">
  <h2>
Introduction
</h2>
  <p>
Content here...
</p>
</section>

<section id="features">
  <h2>
Features
</h2>
  <p>
Content here...
</p>
</section>

Teaching Tip: Demonstrate how browser scrolling behavior changes with anchor links. You can also show how to create a "back to top" link using href="#" or href="#top".

Chapter 6: Images & Media

Adding Images

The <img> element embeds images in web pages:

<!-- Basic image -->
<img src="photo.jpg" alt="Description of photo">

<!-- Image with dimensions -->
<img src="logo.png" alt="Company logo" width="200" height="100">

<!-- Responsive image -->
<img src="banner.jpg" alt="Website banner" style="max-width: 100%; height: auto;">

Important: Always include the alt attribute for accessibility. It describes the image for screen readers and displays if the image fails to load.

Image attributes:

Attribute Purpose
src Source URL of the image
alt Alternative text for accessibility
width/height Dimensions of the image
loading Lazy loading ("lazy" or "eager")
srcset Multiple image sources for different resolutions
sizes Image sizes for different viewports
<!-- Responsive images with srcset -->
<img src="image-800.jpg"
srcset
=
"image-400.jpg 400w, image-800.jpg 800w, image-1200.jpg 1200w"
sizes
=
"(max-width: 600px) 400px, (max-width: 1000px) 800px, 1200px"
alt
=
"Responsive image example"
>

Figures and Captions

The <figure> and <figcaption> elements provide semantic structure for images with captions:

<!-- Figure with caption -->
<figure>
  <img src="chart.png" alt="Sales growth chart 2023">
  <figcaption>
Figure 1: Quarterly sales growth for 2023
</figcaption>
</figure>

The <figure> element can contain:

  • Images
  • Code snippets
  • Videos
  • Audio
  • Diagrams
  • Any content that is referenced from the main content

Teaching Tip: Explain that <figure> is self-contained content that could be moved to another part of the document without affecting the flow. <figcaption> provides a caption that is directly related to the figure.

Image Formats and Optimization

Different image formats serve different purposes on the web:

Format Best For Features
JPEG Photographs Lossy compression, millions of colors
PNG Graphics with transparency Lossless compression, transparency support
GIF Simple animations Limited colors (256), supports animation
SVG Logos, icons, illustrations Vector format, scales infinitely, small file size
WebP Modern web images Better compression than JPEG/PNG, transparency, animation

Image optimization best practices:

  • Choose the right format for the content
  • Compress images to reduce file size
  • Use responsive images with srcset
  • Specify width and height to prevent layout shifts
  • Use lazy loading for offscreen images

Image Maps

Image maps define clickable areas on an image:

<!-- Image with clickable areas -->
<img src="planets.jpg" alt="Solar System" usemap="#solarsystem">

<map name="solarsystem">
  <!-- Circle area -->
  <area shape="circle" coords="100,100,50" href="sun.html" alt="Sun">
  <!-- Rectangle area -->
  <area shape="rect" coords="200,50,300,150" href="earth.html" alt="Earth">
  <!-- Polygon area -->
  <area shape="poly" coords="350,100,400,150,300,150" href="mars.html" alt="Mars">
</map>

Accessibility Note: Image maps can be difficult for users with mobility or vision impairments. Provide alternative navigation methods when using image maps.

Table elements:

Element Purpose

Table Attributes and Spanning

Cells can span multiple rows or columns using rowspan and colspan:

<!-- Table with row and column spanning -->

Name
</th>

</th>
  </tr>
  <tr>
    <th>

</th>
    <th>
Phone
</th>
  </tr>
  <tr>
    <td>

</td>
    <td>

</td>
    <td>
123-456-7890
</td>
  </tr>

Table attributes:

Attribute Purpose
rowspan Number of rows a cell should span
colspan Number of columns a cell should span
headers Space-separated list of header cell IDs
scope Defines scope of header cells (row, col, rowgroup, colgroup)

Teaching Tip: Use physical tables (like a calendar or seating chart) as real-world examples to demonstrate rowspan and colspan concepts.

Accessible Tables

Proper table markup is essential for accessibility:

<!-- Accessible table with scope and headers -->
<table>
  <caption>
Student Grades for Spring Semester
</caption>
  <thead>
    <tr>
      <th id="name" scope="col">

</th>
      <th id="math" scope="col">
Math
</th>
      <th id="science" scope="col">
Science
</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th id="student1" scope="row">
John Smith
</th>
      <td headers="student1 math">
85
</td>
      <td headers="student1 science">

</td>

  </tbody>

Accessibility best practices for tables:

  • Use <th> for header cells
  • Use scope attribute to define header scope
  • Use headers attribute for complex tables
  • Provide a <caption> for table description
  • Use <thead>, <tbody>, and <tfoot> for structure

Chapter 9: Semantic HTML

What is Semantic HTML?

Semantic HTML uses elements that clearly describe their meaning to both browsers and developers:

<!-- Semantic HTML structure -->
<header>
  <h1>
Website Title
</h1>
  <nav>
...
</nav>
</header>

<main>
  <article>
    <h2>
Article Title
</h2>
    <p>
Article content...
</p>
  </article>
</main>

<footer>
  <p>
Copyright information
</p>
</footer>

Benefits of semantic HTML:

  • Accessibility: Screen readers can better interpret page structure
  • SEO: Search engines understand content better
  • Maintainability: Code is easier to read and understand
  • Future-proof: Semantic markup is less likely to break with design changes

Teaching Strategy: Compare semantic HTML to a well-organized book with clear chapters, sections, and headings versus a book where everything is just plain text.

Key Semantic Elements

HTML5 introduced these important semantic elements:

Element Purpose
<header> Introductory content or navigation
<nav> Navigation links
<main> Main content of the document
<article> Self-contained composition
<section> Thematic grouping of content
<aside> Content indirectly related to main content
<footer> Footer for its nearest section
<figure> Self-contained content with optional caption
<figcaption> Caption for <figure>
<time> Machine-readable date/time
<mark> Highlighted text

When to use each element:

  • <header> - Site header, article header, section header
  • <nav> - Main navigation, table of contents, pagination
  • <main> - Primary content (only one per page)
  • <article> - Blog post, news article, forum post
  • <section> - Chapters, tabbed content, grouped content
  • <aside> - Sidebars, pull quotes, advertising
  • <footer> - Site footer, article footer, section footer

Chapter 11: HTML Accessibility

Why Accessibility Matters

Web accessibility ensures that people with disabilities can perceive, understand, navigate, and interact with the web:

  • Visual impairments: Screen readers, high contrast modes, zoom
  • Hearing impairments: Captions, transcripts, visual alerts
  • Motor impairments: Keyboard navigation, voice control, switch devices
  • Cognitive impairments: Clear structure, simple language, consistent navigation

Accessibility benefits everyone:

  • Older users with changing abilities
  • People with temporary disabilities (broken arm)
  • Users in challenging environments (bright sunlight)
  • People using mobile devices with small screens
  • Users with slow internet connections

Legal Requirement: In many countries, web accessibility is required by law for government websites and often for commercial websites as well.

Key Accessibility Practices

Essential HTML accessibility features every developer should implement:

<!-- Proper heading structure -->
<h1>
Main Title
</h1>
<h2>
Section Title
</h2>
<h3>
Subsection Title
</h3>

<!-- Descriptive link text -->
<a href="about.html">
Learn more about our company
</a>
<!-- Instead of: -->
<a href="about.html">
Click here
</a>

<!-- Form accessibility -->
<label for="username">
Username:
</label>
<input type="text" id="username" name="username" aria-describedby="username-help">
<div id="username-help">
Your username must be 6-12 characters long.
</div>

Essential accessibility practices:

  • Use semantic HTML elements
  • Provide text alternatives for non-text content
  • Ensure keyboard accessibility
  • Use proper color contrast
  • Make forms accessible with labels and instructions
  • Ensure content is readable and understandable

  • Efficient structure: Avoid deeply nested elements when possible
